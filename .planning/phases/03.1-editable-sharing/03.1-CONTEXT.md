# Phase 3.1: Editable Sharing - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase adds an editable sharing mode: a shared link that loads document data into the full scheduler app (index.html + app.js) for editing, rather than just viewing in the read-only viewer. The user can then save changes back to the server using the existing upload mechanism.

</domain>

<decisions>
## Implementation Decisions

### URL & Routing
- Editable share link served from the API (like viewer), likely `/edit/{id}` route
- App detects shared-edit mode from URL and fetches document data from API on load
- Reuse the `replaceState(normalizeState(fetchedData))` pattern from onImportJson

### Save-Back Behavior
- Manual save only — user clicks "Upload to API" to push changes (no auto-save)
- PUT to same document ID — overwrites the original shared document
- Show confirmation dialog: "This will update the shared document. Continue?" before overwriting

### Authentication
- API base URL: auto-detected (no manual entry needed when loaded from shared edit link)
- JWT token: still requires manual entry — token is a security credential
- API URL field can be hidden/pre-filled when in shared edit mode

### Claude's Discretion
- API URL auto-detection approach (e.g., `window.location.origin` or similar)
- How to serve index.html from the /edit/{id} route (static file serving or inline)
- Session ID generation for shared edit sessions (to avoid colliding with local data)
- Whether to show a visual indicator that the user is editing a shared document

</decisions>

## Existing Code Insights

### Reusable Assets
- `replaceState(normalizeState(parsed))` — battle-tested pattern for loading JSON into app state
- `onImportJson()` — full flow including route cache reset and UI re-render
- `onUploadToApi()` — upload handler with share URL display and copy button
- Viewer route pattern in Program.cs — serves HTML file for any document ID

### Established Patterns
- State is session-keyed in localStorage: `fieldwork-scheduler-v3:{SESSION_ID}`
- SESSION_ID comes from URL param or auto-generated (app.js lines 250-262)
- 5 MB payload size limit on API
- Bearer token auth via simple string comparison

### Integration Points
- Program.cs: Add `/edit/{id}` route mirroring `/viewer/{id}` but serving index.html
- app.js initialization: Detect `/edit/{id}` URL, fetch and load document data
- app.js upload handler: When in shared-edit mode, use PUT instead of POST
