  <how-to-verify>
**Prerequisites:**
Start the stack locally. Either:
- `docker compose up` (full stack with MongoDB), OR
- `cd backend/ShareService.Api && dotnet run` (dev server — requires MongoDB running separately)

Confirm the API is up: `curl http://localhost:5000/health` should return `{"status":"ok"}`.

**Step 1 — Create a test document**
```bash
curl -s -X POST http://localhost:5000/documents \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $JWT_TOKEN" \
  -d '{"schemaVersion":5,"selectedCityId":"","cities":[{"id":"city_test_001","name":"Test City","schools":[],"days":[],"researchers":[],"routeCache":[]}]}' \
  | python -m json.tool
```
Copy the returned `id` value (e.g., `abc-123-def-456`).

**Step 2 — Open the edit URL in a browser**
Navigate to: `http://localhost:5000/edit/{id}` (replace {id} with the ID from Step 1)

**Step 3 — Verify EDIT-01: App loads correctly (no 404 assets)**
- [ ] Page renders with the full Fieldwork Scheduler UI (not a blank page or raw JSON)
- [ ] Browser DevTools Network tab shows styles.css and app.js load with HTTP 200 (not 404)
- [ ] Page source contains `<base href="/">`

**Step 4 — Verify EDIT-02: Shared data is loaded into the app**
- [ ] The session badge shows `Session: edit_{id}` (not a random session ID)
- [ ] The city "Test City" is present in the city workspace (data from the API was loaded)
- [ ] The app did NOT start with an empty/blank state

**Step 5 — Verify EDIT-04: API URL auto-detection**
- [ ] In the Export/API section, the "API Base URL" field is pre-filled with `http://localhost:5000`
- [ ] The field was filled automatically — user did not type it

**Step 6 — Verify EDIT-03 + EDIT-05: Confirmation dialog + PUT save-back**
- [ ] Rename "Test City" to "Updated City" (or add a new city)
- [ ] Enter a valid JWT token in the JWT Token field
- [ ] Click "Upload to API"
- [ ] A confirmation dialog appears: "This will update the shared document. Continue?"
- [ ] Click Cancel → upload is cancelled, no changes sent
- [ ] Click "Upload to API" again → confirm dialog appears again → click OK
- [ ] Status shows "Shared document updated successfully." (not "Uploaded successfully." or a URL link)
- [ ] No errors in browser console

**Step 7 — Verify persistence**
- [ ] Reload `http://localhost:5000/edit/{id}` in the browser
- [ ] The city now shows "Updated City" (confirming PUT wrote to the API and the data round-trips)

**Step 8 — Verify session isolation**
- [ ] Open `http://localhost:5000/index.html` (or the local file directly) in the same browser
- [ ] Verify the local session does NOT show "Updated City" (edit session is isolated)
- [ ] The session badge on the local version shows a different session ID (not `edit_{id}`)
